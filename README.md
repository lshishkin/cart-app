# React Cart Management SPA

Приложение реализует одностраничный интерфейс для просмотра и редактирования корзин пользователей.

Задиплоино на Versel по адресу (cart-app-two-tau.vercel.app/)[https://cart-app-two-tau.vercel.app/]

### 🧱 Стек технологий
- React (Vite)
- TypeScript
- MUI (@mui/material, @mui/icons-material)
- @tanstack/react-query
- Zustand
- react-router-dom
- @emotion/styled / @emotion/react
- fetch для запросов API

### 📁 Архитектура (FSD)
Проект структурирован по Feature-Sliced Design:

```
src/
├── app/            # корневая область, провайдеры, роутинг
├── entities/       # модели доменных объектов (cart и др.)
├── features/       # специфичные функции (список корзин, редактирование)
├── pages/          # страницы приложения
├── shared/         # повторно используемые утилиты, API, типы, UI
└── index.tsx       # точка входа
```

- **API**: `shared/api/cartsApi.ts` содержит вызовы к DummyJSON
- **Types**: `shared/types/carts.ts`
- **Состояние**: Zustand хранит параметры пагинации
- **React Query** обеспечивает кеширование данных и выполнение мутаций
- **UI**: компоненты используют MUI и стилизуются через emotion

### 🚀 Функционал
1. **Список корзин**
   - пагинация (limit/skip)
   - состояние загрузки/ошибка
   - переход к деталям сохраняет состояние
2. **Детали корзины**
   - отображение товаров и суммы
   - редактирование количества и удаление товара
   - обновление данных и кеша
3. **Глобальный стейт**
   - limit и skip хранятся в Zustand, сохраняются при навигации

### ⚙️ Запуск проекта
```bash
cd cart-app
npm install
npm run dev          # запуск в режиме разработки
npm run build        # сборка
npm run preview      # просмотр собранной версии
```

### 📝 Примечания
- Приложение разворачивается на Vercel/Netlify как статический сайт (dist).
- Для деплоя достаточно скопировать `dist` или настроить генерацию через платформу.
